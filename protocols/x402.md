# x402 — HTTP Payment Protocol

**Creator:** Coinbase (now x402 Foundation)
**Repo:** [github.com/x402-foundation/x402](https://github.com/x402-foundation/x402) (canonical; coinbase/x402 is now a dev fork)
**Version:** Protocol V2 (production SDKs in TypeScript, Python, Go v2.9.0)
**License:** Apache 2.0
**Foundation:** x402 Foundation (co-founded with Cloudflare, Sep 2025). Foundation charter published Apr 2026.

## What It Is

Settlement layer for per-request agent payments using HTTP 402 status code. An agent hits an API, gets a 402 response, signs a stablecoin authorization (EIP-3009 transferWithAuthorization), attaches it to the retry, and the payment settles on-chain. No accounts, no API keys, no invoices.

## Core Concepts

- **402 Response** — server returns payment requirements (amount, recipient, chain)
- **Facilitator** — verifies the payment permit and settles on-chain. Coinbase runs one, but anyone can run their own.
- **Schemes** — `exact` (pay the stated amount) and `upto` (authorize up to X, settle actual usage)
- **Multi-chain** — Base, Ethereum, Polygon, Solana, Algorand, Aptos, Hedera, Stellar, Sui, Keeta, Stable testnet
- **Upto scheme** — variable/max pricing (authorize up to X, settle actual usage). Shipped in TS and Go SDKs.

## What It Covers

- Per-request micropayments (agent pays per API call)
- On-chain settlement with gasless signing (EIP-3009)
- Facilitator architecture (verification + settlement)
- MCP transport (payment signaling in MCP tool calls)
- A2A transport
- Multi-chain support across EVM and non-EVM

## What It Doesn't Cover (Explicitly Out of Scope)

- Client-side budget management
- Agent delegation or mandates
- Cross-service spending coordination
- Subscription/recurring payments
- Fiat payment rails (crypto-only currently, fiat planned)

## Relationship to Other Protocols

- **Competes with MPP** at the settlement layer (stablecoin-only vs multi-rail)
- **Plugs into AP2** via the A2A x402 extension (crypto settlement for AP2 mandated purchases)
- **Independent of ACP/UCP** at the commerce layer
